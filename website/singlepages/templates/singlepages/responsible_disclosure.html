{% trans "Version" %} 1.1, 16-11-2020

{% blocktrans trimmed %} At Thalia, we take security of our infrastructure and digital systems seriously. We develop, build and deploy all systems with security in mind. However, we can never be 100% secure and vulnerabilities will, unfortunately, present themselves. If you find a (potential) vulnerability or security issue, we would appreciate it if you inform us as soon as possible. {% endblocktrans %}

{% trans "How to notify us" %}

  • {% trans "Send us any and all reports about potential security issues to www@thalia.nu (in either English or Dutch)." %}
  • {% trans "Please also consider" %} https://thalia.nu/.well-known/security.txt.

{% trans "We ask you to" %}

  • {% trans "give detailed reports of your findings to help us identify the underlying problem." %}
  • {% trans "only exploit vulnerabilities to confirm their presence." %}
  • {% trans "not share your report with anybody else until the problem has been fixed." %}
  • {% trans "involve us if you want to publish your report once it has been fixed." %}
  • {% trans "respect the privacy of our members." %}

{% trans "We promise you to" %}

  • {% trans "take your reports seriously." %}
  • {% trans "respond to your reports as soon as possible." %}
  • {% trans "solve the issue as soon as possible." %}
  • {% trans "respect your privacy and treat your reports confidentially." %}
  • {% trans "keep you up to date and work with you to solve the underlying problem." %}
  • {% trans "credit you in any internal and external announcements (with your permission)." %}
  • {% trans "not take any legal action if you followed the above guidelines." %}

{% trans "What is the scope" %}

  • {% trans "You may try to exploit: https://thalia.nu/*" %}
  • {% trans "Any resource that is not listed here is also prohibited." %}

{% trans "If you happen to find something on an unlisted resource, please let us know." %}

{% trans "Which attacks are disallowed" %}

{% trans "Because we are a small organisation we ask you to not use the following attack methods:" %}

  • {% trans "social engineering" %}
  • {% trans "physical attacks" %}
  • {% trans "(distributed) denial of service" %}
  • {% trans "spam attacks" %}
  • {% trans "automated scanners" %}

{% trans "Exceptions" %}

{% trans "The following reports will not be taken into consideration:" %}

  • {% trans "the policy of SPF / DKIM / DMARC" %}
  • {% trans "the policy of CSP" %}
